# DEVA方式 BEN2/TB ハイブリッド背景除去アプリ 実装計画

- 作成日: 2026-07-01
- 対象アプリ: `gradio_app_sam2_ben2_tb_hybrid_deva_for_Movie.py`
- ベース: `gradio_app_sam2_ben2_route_a_deva_for_Movie.py`
- 区分: 実装計画 + 実装後記録
- 目的: GroundingDINO/SAM2.1 が追跡した人物領域のみ `transparent-background` を使い、それ以外を BEN2 で処理して最終 alpha を合成する

---

## 1. 背景と狙い

BEN2 は画面の主体となる被写体を大きく捉える力が強い一方で、髪の毛などの繊細な表現は苦手になりやすい。

反対に `transparent-background` は人物や髪の毛の切り抜きに強い。そこで本アプリでは、両モデルを「どちらか一方」ではなく、領域ごとに役割分担する。

- 人物領域: `transparent-background`
- 人物領域外: BEN2
- 領域決定: GroundingDINO の text prompt → SAM2.1 画像 mask → SAM2.1 動画伝播 → consensus
- 最終出力: 2 系統 alpha を選択式の合成方式で統合

---

## 2. フロー

```text
Input Video
  |
  v
VideoReader
  |
  v
DevaSemiOnlineTracker
  - GroundingDINO(text -> bbox)
  - SAM2 image mode(box -> mask)
  - SAM2 video propagation
  - consensus merge / missed cleanup
  |
  | masks: {frame_masks, per_object_logits, metadata}
  v
BEN2TransparentHybridVideoExtractor
  |
  +--> BEN2 alpha（全画面、主被写体/非人物側）
  |
  +--> transparent-background alpha（SAM2人物mask内のみ）
  |
  v
Hybrid alpha composition
  - lighten / 比較明
  - person_over_ben2
  - ben2_over_person
  - screen
  |
  v
VideoWriter / FrameSequenceWriter / TrackingOverlayWriter
```

---

## 3. Haystack Component 境界と I/O 契約

| Component | 責務 | 入力 | 出力 |
| --- | --- | --- | --- |
| `VideoReader` | 動画を RGB frame list と metadata に変換 | `video_path`, `max_frames`, `frame_step` | `frames:list`, `metadata:dict` |
| `DevaSemiOnlineTracker` | 検出島 + SAM2伝播 + consensus を内部に閉じる | `frames`, `metadata`, `text_prompt`, 閾値類 | `masks:dict` |
| `BEN2TransparentHybridVideoExtractor` | BEN2/TB 推論と alpha 合成、streaming 出力 | `frames`, `masks`, `metadata`, 合成設定 | `matte:dict` |
| `VideoWriter` | matte の動画書き出し | `matte` | `matte` |
| `FrameSequenceWriter` | matte の PNG 連番書き出し | `matte` | `matte` |
| `TrackingOverlayWriter` | 追跡 mask の確認動画 | `frames`, `masks`, `metadata` | `overlay:dict` |

重要な契約:

- `masks["frame_masks"]` は `source_index -> (H,W) float32/bool`。
- `BEN2TransparentHybridVideoExtractor` は `frame_masks[source_index]` を人物領域として扱う。
- `frame_masks` が空または全 0 の frame では `transparent-background` を実行せず、BEN2 のみで処理する。
- 出力 `matte` は既存 writer が受け取れる `rgba_video_path`, `alpha_video_path`, `preview_video_path`, sequence dirs を持つ dict。
- 重いモデル初期化は import 時に行わず、Component の `run()` / 内部 `warm_up()` で遅延する。

## 4. Alpha 合成方式

本アプリの `Alpha 合成方式` UI は、奥行き推定モデルを入れずに手動で重ね方を選ぶためのもの。

| UI | 内部値 | 使いどころ |
| --- | --- | --- |
| `lighten / 比較明（推奨）` | `lighten` | 画素ごとに明るい alpha を採用。黒抜けを避ける初期値。 |
| `person_over_ben2（人物TBを手前）` | `person_over_ben2` | 人物と髪を BEN2 より優先したい場合。 |
| `ben2_over_person（BEN2を手前）` | `ben2_over_person` | BEN2 の主体把握を優先したい場合。 |
| `screen（スクリーン）` | `screen` | 両 alpha を柔らかく加算的に残したい場合。 |

実装上は、まず BEN2 alpha と TB alpha を full alpha として合成し、その後 `person_region` で空間 blend する。これにより feather 境界で両 alpha が不透明なのに半透明 halo になる問題を避ける。

## 5. ルートA案との関係

`計画書/2026-06-22_動画αマット_ルートA案_ブラー誘導_仕様書.md` の「SAM2由来のマスク領域以外にブラーをかけ、背景除去モデルが余計なものを検出しにくくする」施策は、既存 RouteA 実装に入っている。

実装箇所:

- `pipelines/components/ben2_components.py`
  - `_process_union_frame()`
  - `mask` ありの場合に `dilate_mask_to_gate(mask, dilation_px)` でゲートを作る
  - `blur_background_outside_gate(image_rgb, gate, blur_kernel, blur_sigma, feather_px)` でゲート外をブラーする
  - ブラー済み `guided` を BEN2 に渡す
- `pipelines/components/route_a_common.py`
  - `blur_background_outside_gate()` が「ゲート G の外側を強ブラーし、被写体をシャープに保った誘導フレーム I' を作る」責務を持つ

今回の BEN2/TB ハイブリッドは RouteA のブラー誘導そのものではなく、役割分担型の別 alpha 生成 Component として追加した。既存 RouteA は維持し、比較できるようにしている。

## 9. 注意点

- UI fixed として記録するには Playwright 実動画検証が必要。
- `segment-anything/` と `samurai/` は直接変更しない。
- `torch.load(..., weights_only=False)` は使わない。
- `try/except: pass` は使わない。
- 長尺・4K・多数人物では DEVA tracker 側の `per_object_logits_max_side` による RAM 制御を維持する。
- `transparent-background` は人物 mask が空の frame では実行しない。全画面 TB 実行は要件外かつ重いため禁止。
