# 2026-06-05 Copilot指示体系 多層化リファクタ計画

## 1. 背景

`資料/データサイエンティストのためのAGENTS.mdとSkills.md` の設計思想（薄い共通指示 + ルーター + 作業別 skill）を、Matting-Anything の実情に合わせて最小スコープで導入する。

現状課題:

- `.github/copilot-instructions.md` が肥大化し、常時ロード時のトークン効率が悪い。
- `copilot-instructions.md` と `workflow.instructions.md` の重複が多い。
- ルーティング専用の `AGENTS.md` が無い。

## 2. 採用方針（確定）

- Aスコープ（最小）を採用。
- 追加で再利用 prompt を数個導入する。
- Gradio 5 専用 skill と SAM2 系 skill を新設する。
- `.venv + pip` 運用を維持し、uv には移行しない。
- `ERROR_LOG.md` / `WHITEBOARD.md` / `REFERENCE.md` はルート据え置きとし、AGENTS.md から参照する。

## 3. 実施内容

### Phase 1: ルーター導入 + 共通指示の薄化

1. ルートに `AGENTS.md` を新規作成。
2. `.github/copilot-instructions.md` を「常時適用の短いルールカード」に再構成。
3. 最重要ルールは `AGENTS.md` と `copilot-instructions.md` の両方に保持する。

### Phase 2: 領域 skill の分離

1. `.github/skills/gradio5-sam2-ui/SKILL.md` を新設。
2. `.github/skills/sam2-tracking-dino/SKILL.md` を新設。
3. 既存 `.github/skills/haystack-pipeline/SKILL.md` は継続利用。

### Phase 3: workflow 重複解消

1. `.github/instructions/workflow.instructions.md` から領域詳細の重複記述を削減。
2. 実行フロー・レビュー運用・記録更新ルールは維持。

### Phase 4: 再利用 prompts 追加

1. `.github/prompts/plan-change.prompt.md`
2. `.github/prompts/verify-ui-playwright.prompt.md`

※ 既存 skill/agent と競合するため、review-diff / update-error-log prompt は追加しない。

### Review 1: Copilot指示体系整合性

- 設計思想（薄い共通指示 + ルーター + skill分離）に整合。
- prompts frontmatter は VS Code 標準の `mode: agent` を採用。

### Review 2: 私たちの整合性

- ERR の割当漏れを補正し、全38件のカバレッジ方針を明文化。
- 最重要ルールの常時参照性を維持するため、REMINDER を削除しない。

### Review 3: 過剰すぎないか

- prompts は 4 個案から 2 個に縮小。
- docs/agent 新設、uv移行、CI scripts 追加は今回は見送り。
