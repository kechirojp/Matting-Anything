# 2026-06-17 動画アルファ処理フロー logit保持リファクタ実装計画

> 対象: 動画パイプライン（DINO BBOX → SAM2 → transparent-background）
> 範囲: 動画系のみ（`for_Movie` + `video_*` components）。静止画版・深度推定は対象外。

## 1. ゴール

資料のフロー図（logit 保持版・深度除外）に沿って、動画パイプラインを
「per-object logit 保持 → ピクセル softmax 所有権 → 連続アルファ合成」へ作り直す。
最重要の注意点（box + 補正点を同一 `obj_id` の 1 回呼び出しで結合）を確実化する。

## 2. 確定事実（Web / ソース確認済み・ハルシネーション防止）

- SAM2 `add_new_points_or_box`: box と points を**同一 obj_id の 1 回呼び出しで結合可能**。
  `box` 指定時は `clear_old_points=True` 必須 ＝ box 後付け不可。現行コードは box + 割当点を
  1 回で渡しており既に正しい形。
- `propagate_in_video` は **obj ごとの mask logit を yield** する。per-object 取得可能。
- SAM2 内蔵 `non_overlap_masks` は argmax 的ハード解決。フロー図③は softmax（連続）のため手実装が必要。
  背景 logit 基準は **0**（SAM2 判定境界、logit > 0 = 前景）。
- transparent-background `Remover.process` は**画像のみ入力・マスク入力不可**。
  マスクガード（dilate / feather / soft）は extractor 側で適用する。

## 3. スコープ（ユーザー確定）

- tb 適用: 段階導入。
  - Phase1 = 1 フレーム 1 回 tb × 所有権ゲートで継ぎ目線根治。
  - Phase2 = 対象ごと crop tb。
- 背景 logit = 0 基準で softmax。温度 τ は config 化（ハードコード禁止）。
- UI は維持しつつ、検証用ヘッドレス CLI 実行経路を追加。

## 4. Phase 1: per-object logit 契約 + 所有権解決 + 連続合成

1. `SAM2VideoPropagator.run`: `np.maximum` union を廃止し per-object logit を emit。
   出力契約は `masks = {"per_object_logits": {frame_idx: (N,H,W) float32}, "frame_masks": {}}`。
   box + 点結合（最重要点）は維持し明示テストを追加。
2. 新規 Component `OwnershipResolver`（`pipelines/components/ownership_resolver.py`）:
   入力 per-object logits + 背景チャネル（logit=0）→ 温度 τ softmax → 所有権 weight。
   出力 `ownership = {frame_idx: (N+1,H,W) float32}`（最終チャネル = 背景）。
3. `TransparentBGExtractor` / `TransparentBGVideoExtractor`: ownership を soft guard として受理し、
   crop → `Remover.process` → full_alpha → guard 乗算でアルファ補正。
4. パイプライン `sam2_tb_video_pipeline.py` に `OwnershipResolver` を propagator ↔ extractor 間へ挿入。
5. movie app callback で τ を config から渡す。UI 維持。

## 5. Phase 2: 対象ごと crop tb 合成（④⑤ 忠実版）

- 対象ごとに crop → tb → 連続 alpha、所有権で合成。硬物体は SAM2 sigmoid + 小 dilation 代用。
- matting 要否フラグ設計。コスト最適化（crop 限定）。Phase1 GREEN 後に着手。
