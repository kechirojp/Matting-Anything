## 1. 実コード精査で確定した事実（根拠付き）

### F1. guard 乗算が tb の連続アルファを領域全体で削っている（S2 の真因）
- `pipelines/components/model_components.py` `TransparentBGExtractor.run`:
  ```python
  full_alpha = full_alpha * guard
  ```
- union モードでは `OwnershipResolver` が `frame_masks` を **前景 soft = 1 − 背景所有権（領域全体の連続確率）** に差し替える（`ownership_resolver.py`）。
- その float mask が `TransparentBGExtractor.run` に渡り、float のため `soft_probability_guard`（`common.py`）を経由する。これは **領域内部も 1.0 未満の連続値**になり、tb の綺麗な人物アルファ内部に乗算されて**人物が半透明化**する。
- ユーザーの指摘「**領域全体**の信頼度をアルファに変えている／本来は**境界**の信頼度」「**拡張ならわかるが** guard が削るのは論外」は実コードと一致。
- 本来 guard が担うべき役割: **mask 形状の外側に漏れた alpha を 0 にするゲート**（= 二値 dilate で形状外のみ 0、内部は 1.0）。内部の連続値乗算は設計誤り。

### F2. per_object 合成式が積ゼロで黒化・多 box で停止（S5, S6 の主因）
- `video_model_components.py` `_run_per_object_frame`:
  ```python
  alpha_final = composite_alpha_by_ownership(per_object_alphas, ownership)  # Σ ownership_o × tb_alpha_o
  ```
- 対象 o の所有権領域（SAM2）と tb_alpha_o（salient matting の結果）が空間的にズレると積 ≈ 0 → 全画素黒（S5 の真っ黒）。
- tb 呼び出しが frames × objects 回。多 box（長文字プロンプト）で計算量が膨らみ途中停止（S6）。
- **2026-06-18 に既定を `per_object` にしたのは時期尚早だった**ため、暫定で `union` に戻し済み（本計画とは別の安全是正）。ユーザー方針「**まず per_object 毎の背景除去が成功しなければ union もできない**」に従い、Phase 2 で per_object 経路自体を作り直す。

### F3. フレームサンプリングが常に先頭固定（S7 の真因）
- `video_common.py` `sample_frame_indices`:
  ```python
  return list(range(0, int(frame_count), int(frame_step)))[: int(max_frames)]
  ```
- 常に index 0 から最大 `max_frames` 枚。途中フレームを起点にしても先頭 30 枚しか読まれないため `prompt_frame_idx` は 0〜(max_frames−1) でしか機能しない。
- 仕様要件: 初期推論は 30 枚程度に抑えつつ、**任意フレームを起点に前後双方向へ追跡**できること。ユーザー言「初期推論時の条件分岐も不要」。

### F4. box+point 逐次結合は実装済み（S9 は OK）
- `video_model_components.py`: 単一 box + points は 1 回の `add_new_points_or_box(box=, points=, labels=)` で同梱。複数 boxes 時は `assign_points_to_boxes` で各 point を最近傍 box の同 obj_id に同梱。
- 資料§2「box → 同じ obj_id にポジ/ネガ点を追加 → propagate」を満たす。**ここはバグではない**。

### F5. SAMURAI は既定で未使用
- tracker 既定 `sam2_hiera_l`（素の SAM2）。SAMURAI は `INFERENCE_TRACKER_VARIANT=sam2_samurai` 設定時のみ可視・選択可。

### F6【未確定・要調査】 SAM2 が前景でなく背景を掴む理由（S3, S4, S5）
- 「画面全体 box は背景を掴みやすい」という説明は**実測に基づかない憶測**であり撤回する。
- ユーザーの反証仮説:「**point prompt が正しく効いていれば背景は掴まないはず**」。実コード上 point は SAM2 に同梱されている（F4）が、**実際に SAM2 が受け取った prompt と出力 mask の対応は未計測**。
- → Phase 4 冒頭で **ヘッドレス CLI による計測**を行い、真因（prompt 座標系のズレ／label の正負反転／out_obj_ids の取りこぼし／overlay と最終 mask の派生差など）を**事実として特定してから**修正方針を確定する。

## 2. 改修方針（Phase 1→4）

### Phase 1: guard を「拡張ゲート」に正す（S2 解消・最優先）
- **目的**: tb の連続アルファ内部を削らない。guard は mask 形状**外**の漏れだけを 0 にする。
- **方針**:
  - `TransparentBGExtractor.run` の guard 生成を、mask が float（soft）でも **二値化 → dilate した二値 guard（内部 1.0・外部 0）** を既定とする。
  - soft（連続値）guard は `mask_guard_feather > 0` のオプトイン時のみに限定する（現状の「float なら必ず soft guard」を廃止）。
  - これにより「2値化しない（logit 保持）」方針は SAM2 mask 伝搬・所有権解決の段階で維持しつつ、**最終アルファの連続性は tb 由来で確保**し、guard は形状ゲートに徹する。
- **テスト（RED→GREEN）**:
  - float soft mask を渡したとき、mask 内部画素の最終 alpha が tb の crop alpha と一致（guard で減衰しない）こと。
  - mask 形状外（dilate マージン外）の漏れ alpha が 0 になること。
  - `mask_guard_feather > 0` 指定時のみ境界が feather されること。
- **影響範囲**: `model_components.py`（guard 分岐）。`common.py` は触らず既存関数を使い分ける。

### Phase 2: per_object 背景除去を成功させる（S1, S5, S6 の本丸）
- **前提**: ユーザー方針「per_object が成功して初めて union が成り立つ」。
- **目的**: 対象ごとに tb を成功させ、所有権で**破綻なく**合成する。積ゼロ黒化を解消する。
- **調査 → 設計（Phase 2a）**:
  - per_object で黒化する条件を計測（ownership と tb_alpha の空間ズレ、対象が背景を掴むケースの切り分け）。
  - 合成式の見直し候補: `Σ ownership_o × tb_alpha_o`（積）→ 破綻時の代替（例: 所有権で**領域を選ぶ**ゲートにして tb_alpha をそのまま採用、または `max` 合成）を検討。Phase 4 の prompt 真因と連動。
- **実装（Phase 2b）**: 合意した合成式で per_object を作り直し、多 box でも停止しない（計算量・進捗・メモリ）ことを担保。
- **テスト**: 合成数学のユニット（積ゼロ回帰）、extractor モックで N 回呼び出し・RGB 保持・黒化しない合成を検証。
- **既定の扱い**: Phase 2 完了・GREEN 確認まで config 既定は `union` を維持。成功確認後にユーザー判断で既定切替。

### Phase 3: 任意フレーム起点・双方向追跡（S7）
- **目的**: 初期推論は最大 30 枚程度に抑えつつ、任意フレームを起点に前後双方向へ追跡。先頭固定をやめる。
- **方針**:
  - `sample_frame_indices` を起点フレーム中心のウィンドウ（前後）でサンプリングできるよう拡張。`prompt_frame_idx` をサンプリング窓の中央付近にマップ。
  - 起点に依存する条件分岐（「初期推論 30 以下のときだけ」等）を撤廃し、常に同一ロジックで動かす。
  - `prompt_frame_idx` の範囲検証を新サンプリングに合わせて更新（UI fail-fast も）。
- **テスト**: サンプリング窓の中央起点・端起点・双方向で、起点フレームが必ず含まれ、前後フレームが追跡対象になることを検証。

### Phase 4: point 受け渡し真因の調査と背景トラッキング修正（S3, S4, S5）
- **ハルシネーション対策**: 修正前に**必ず計測**。憶測で実装しない。
- **調査（Phase 4a）**: `run_video_matting_headless.py` で再現データを用い、
  - SAM2 に渡る points/labels/box の座標系・正負ラベル・obj_id を実値ログ。
  - prompt_frame の単フレーム推論結果 mask を保存し、前景/背景どちらを掴んでいるか確認。
  - overlay（union soft）と最終 frame_masks の派生差を突き合わせ、解離点を特定。
- **修正（Phase 4b）**: 計測で確定した真因のみを修正（例: 座標スケール不一致、label 既定の誤り、out_obj_ids 取りこぼし等）。
- **テスト**: 計測で見えた不整合に対する回帰テストを追加。

## 4. リスクとロールバック

- Phase 1 は guard 分岐のみ変更。回帰時は `mask_guard_feather` 既定 0 のまま旧分岐へ戻すだけで復旧可能。
- Phase 2 は per_object 経路に限定。union 経路（既定）は不変のため、失敗しても既定動作に影響しない。
- Phase 3 はサンプリング関数の拡張。旧 `range(0, n, step)[:max]` を「起点 0・双方向 OFF」で再現できる後方互換を保つ。
- Phase 4 は調査主導。修正は計測で確定した最小差分のみ。
