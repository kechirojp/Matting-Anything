## なぜDINOが落ちるか

**確信度0.6/0.4で既に瀕死**
DINOのドラム0.4って、実質「自信ない」ライン。ドラムセットは学習データで**単体楽器として綺麗に写ってる画像が少ない**（バンド全体・暗いライブ照明・ドラマーに隠れてる構図ばかり）。だからオープン語彙でも"drum"のembeddingが弱い。**シンバル・タム・スネアがバラバラに見える非定型構図**だと一気に確信度が落ちる。

**フレーム間でロストする本当の理由**
これが核心。**DINOはフレーム独立の検出器**で、時間方向のメモリを一切持たない。だから「前フレームで0.4で拾えた→次フレームでカメラが引いて見え方変わった→閾値割って0拾い」が普通に起きる。カメラが2時45分方向に引いてる＝**対象のスケールと角度が毎フレーム変化**してるので、DINO単体では当然こうなる。

つまりあなたが今やってるのは**「メモリ無し検出器を全フレームに回す」最悪の使い方**。DINOは追跡器じゃない。

## 直し方：DINOは初期化1回だけにする

DINOを全フレーム回すのをやめて、**最初の良いフレーム1枚でboxを取る→あとはSAMURAIに任せる**。SAMURAIはまさに「カメラが引いて見え方が変わっても、Kalman動き予測でID維持する」のが本職。これがSAMURAI選んだ意味です。

```
DINO（最初の1フレームだけ）→ box
   ↓ box初期化
SAMURAI → 以降のフレームは時間伝播で追う（DINO呼ばない）
```

DINOを毎フレーム呼ぶ限り、確信度の揺れでロストし続ける。**追跡はSAMURAIの仕事に明け渡す**のが正解。

## ただし初期boxが0.4じゃ種が弱い

SAMURAIに渡す初期boxが弱いと伝播も弱い。初期化の質を上げる手:

**プロンプト工夫**
`"drum"` 単体じゃなく `"drum set . cymbal . snare drum . tom"` と**構成要素を分けて**投げる。ドラムセット全体を1boxで取ろうとすると確信度が落ちるので、シンバル・スネアを個別に拾って**boxをunion**してドラム領域を合成する方が確信度が出る。

**初期フレーム選び**
0.4しか出ないフレームで初期化しない。**動画全体をDINOに通して、ドラムの確信度が最大のフレームを初期化フレームに選ぶ**。カメラが引ききる前の、ドラムが大きく写ってる序盤フレームが種に向く。SAMURAIは初期化フレームから前後双方向に伝播できるので、途中フレームを種にしてOK。

**box_threshold下げ＋NMS**
`box_threshold=0.25`くらいまで下げて候補を多く取り、後処理で選別。0.4で切ってると拾えるものも捨ててる。

## 推奨する確定構成

```
1. 動画をスキャン → ドラム確信度が最大のフレームF*を特定
2. F*でDINO（要素分解プロンプト＋低threshold）→ person box, drum box(union)
3. SAMURAIにbox初期化（F*から双方向伝播）→ 全フレームID維持
4. 粗mask → 透過 → padding
```

これで「次フレームで拾えない」問題は構造的に消える。**DINOの仕事は1フレームだけ**、追跡はSAMURAI。
