# **0. プロジェクト概要**

---

動画の中から、ねらった対象（たとえば「ギターを弾く人」）を **言葉で指示するだけできれいに切り抜き、背景を透明にする** 仕組みを作っています。

今回は、**前回（6/16）の改修方針への対応結果**、**今できあがっている切り抜きの流れ（フロー）の全体像**、そして**見つかった弱点をどう直すか**を、図でわかりやすくご報告します。

## **2. 詳細**

### **2.1 今までのフロー（処理の全体像）**

結論：**「言葉で指示 → 探す → 追いかける → 型紙を作る → 背景を消す → 重ねて仕上げる」** の6段階で動いています。

複数のAI（自動で判断するプログラム）を部品のように順番につなげています。たとえるなら、**工場のベルトコンベアで少しずつ加工して完成品にする**イメージです。

```text
【図1：今までのフロー】

  ┌──────────────────────────────────────┐
  │ ① 言葉で指示：「ギターを弾く人」と入力                  │
  └──────────────────────────────────────┘
                       │ 言葉
                       ▼
  ┌──────────────────────────────────────┐
  │ ② 画面を見て、対象を四角い枠で囲む（DINO）             │
  └──────────────────────────────────────┘
                       │ 四角い枠
                       ▼
  ┌──────────────────────────────────────┐
  │ ③ 追跡AIを2つ使って、対象を追いかける（番号をつけて追跡）  │
  └──────────────────────────────────────┘
                       │ 四角い枠
                       ▼
  ┌──────────────────────────────────────┐
  │ ④ 切り抜きの「型紙」を作る（SAM2.1）                 │
  └──────────────────────────────────────┘
                       │ 型紙
                       ▼
  ┌──────────────────────────────────────┐
  │ ⑤ 背景除去AIで背景を消す                          │
  │     ・人    → transparent（髪などの細かさが得意）     │
  │     ・ギター → BEN2（物の取り出しが得意）            │
  └──────────────────────────────────────┘
                       │ 切り抜き結果
                       ▼
  ┌──────────────────────────────────────┐
  │ ⑥ 3つを重ねて1枚にまとめる                        │
  │     ・人  ・ギター  ・型紙（チラつき止めの土台）        │
  └──────────────────────────────────────┘
                       │ 完成
                       ▼
                 ★ 透明動画（完成）
```

補足説明：

- **DINO（ディーノ）**：言葉で指示された場所を画面から探し、四角い枠で囲むAI。以降は「枠づくりAI」。
- **追跡AIを2つ使う理由**：確実な四角い枠を作って型紙づくりAIへ渡し、**型紙を確実に作る**ためです。
- **型紙**：どこを残し、どこを消すかを示す白黒の下書き。

そして背景除去AIは、得意分野で担当を分けています（下表）。

| **背景除去AIの使い分け** | **transparent（人向け）** | **BEN2（物向け）** |
| --- | --- | --- |
| 得意なこと | 人物、とくに髪の毛などの細かい部分 | 画面の主役（物）をうまく取り出す |
| 苦手なこと | 人以外を間違えやすい・動画でチラつきやすい | 髪の毛などの細かい部分 |

### **2.2 見つかった弱点 — 型紙が意図しないところで「細切れ」になる**

結論：**型紙づくりAI（SAM2.1）が、本来つながっているのはずの型紙を、バラバラの破片に分けて出してしまう**ことがわかりました。

ただし、**全体の形と輪郭は合っています**。たとえるなら、**正しい絵なのに、ジグソーパズルがバラけて並んでいる**ような状態です。バラけたままでは後の工程がうまくつながりません。

---

### **2.3 古くて確実な技術で直す（新しいフロー）**

結論：**新しいAIは足さず、長年使われてきた「枯れた（＝古くて安定した）画像処理」だけで2か所を補強**します。

1. **型紙を作った直後** … バラバラの型紙を**ひとつにまとめる**。
2. **背景を消したあと** … 仕上げとして3つの補正を入れる。
    - **領域内の半透明をなくす**（本来くっきり残すべき内側のモヤモヤを消す）
    - **コマごとのチラつきを止める**
    - **輪郭の細部を元の映像から復元する**

```text
【図2：枯れた技術を足した新フロー】

  ┌──────────────────────────────────────┐
  │ ①〜④ 言葉で指示 → 認識 → 追跡 → 型紙づくり（今までと同じ）  │
  └──────────────────────────────────────┘
                       │ 型紙（※バラバラの細切れ）
                       ▼
  ┌──────────────────────────────────────┐ ★追加
  │ ★ 細切れの型紙を、ひとつにまとめる                  │
  └──────────────────────────────────────┘
                       │ きれいな型紙
                       ▼
  ┌──────────────────────────────────────┐
  │ ⑤ 背景除去AIで背景を消す（今までと同じ）             │
  │     ・人 → transparent   ・ギター → BEN2         │
  └──────────────────────────────────────┘
                       │ 切り抜き結果
                       ▼
  ┌──────────────────────────────────────┐ ★追加
  │ ★ 仕上げ補正                                  │
  │     ・領域内の半透明をなくす                       │
  │     ・コマごとのチラつきを止める                    │
  │     ・輪郭の細部を元画像から復元する                 │
  └──────────────────────────────────────┘
                       │
                       ▼
            ★ 透明動画（チラつきの少ない完成版）

  ────────────────────────────────────────
  古くて確実な技術を選んだ理由
   ・10年以上使われてきた、枯れて安定した画像処理だけで作れる
   ・新しいAIを足さないので、仕事で使ってよいか（権利）の心配がない
   ・動きが予測でき、結果が安定する（確実性が高い）
  ────────────────────────────────────────
```

判断：**新しいAIを増やすと、ライセンスの確認や動作の不安定さが増えます**。今回は確実さを優先し、**枯れた技術だけで直す**方針にしました。

参考までに、型紙づくりAI（SAM2.1）が出す「細切れの型紙」を図にすると、次のようなイメージです。

```text
SAM2.1の出力。同じ物体なのにフラグメントが分離している。
█=マスク（残す部分）  ・=背景

   ・・█・・・███・・・・█・・
   ・██・・・・█・・██・・・・    ← 隙間(ギャップ)で分断
   ・・██・・・・・██・・・・・
   ・・・███・・██・・・・・・    ← 内部に穴(hole)も存在
   ・・・・██・██・・・・・・・
   ・・・・・████・・・・・█・    ← 孤立した小ノイズ
```

直したい問題点は3つです。

1. **ギャップ** … フラグメント（破片）どうしの隙間
2. **穴** … 領域内部の抜け
3. **ノイズ** … 無関係な小片

具体的に使う技術は、以下のとおりです（すべて昔からある定番の画像処理です）。

| 工程 | 使う技術 | ごく短い説明 |
| --- | --- | --- |
| 前処理：AIに渡す前の下ごしらえ | 適応的ヒストグラム平坦化（CLAHE） | 元動画をコピーして「型紙づくり専用」の映像を作り、全コマの明暗のメリハリを整える。AIが対象を見分けやすくなる（元の映像はそのまま残すので見た目は変わらない） |
| 細切れの型紙をまとめる | 膨張・収縮（モルフォロジー処理） | 型紙を少し太らせて隙間をふさぎ、元に戻す。バラけた破片をつなげる |
| 〃 | 穴埋め（ホール塗りつぶし） | 型紙の内側にできた小さな抜け穴をふさぐ |
| 〃 | 最大領域の抽出 | いちばん大きいかたまりだけ残し、ゴミのような小片を捨てる |
| 仕上げ：半透明をなくす | しきい値処理（トリマップ） | 「残す／消す」をはっきり分け、内側のモヤモヤを消す |
| 仕上げ：チラつき止め | フレーム間の平均化（時間的な平滑化） | 前後のコマと見比べてならし、コマごとのゆれを抑える |
| 仕上げ：輪郭の復元 | ガイデッドフィルタ | 元の映像を手本に、ぼやけた輪郭の細部をくっきり戻す |

---

## **3. 次のアクション**

1. **（最優先）細切れの型紙をひとつにまとめる手当てを実装**し、後の工程へきれいな型紙を渡せるようにする。
2. **仕上げ補正（半透明の排除・チラつき止め・輪郭の復元）を実装**し、透明動画のチラつきを減らす。
3. 実際の映像で効果を確認し、**強さの数値（どれくらいまとめる／どれくらい補正する）を調整**する。
4. **【前回からの残タスク】処理時間・AIサイズの調査に着手**する（マスクの改修を優先したため、まだ未着手です）。
5. **【前回からの残タスク】奥行きで前後を判断して重ねる方式の事前計測**（処理速度・メモリ）を行い、採用するか判断する。
