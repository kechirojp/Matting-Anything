1. **3 問題はすべて「同一の巻き戻り」が原因**。現在の作業ツリーの動画 UI ファイルは、UI/callback 配線が欠落した **退行版（503 行）**であり、これは `調査/2026-06-04_222747_現行動画パイプライン_フロー調査_MAM比較.md` が解析した壊れた版と同一。

つまり `2702d6b` の後に「Gap A-D の洗練 + ERR036-038 ランタイム修正」を行った RED テストだけが残り、その**実装本体は一度もコミットされず失われていた**（`git stash list` / `git branch` / `git reflog` いずれにも痕跡なし → 再実装が必須）。

### 7.4 git stash 誤用と完全復旧（重要・透明性のため記録）

退行の切り分け中に `git stash` を実行したが、作業ツリーには**ユーザーの広範な未コミット作業**（多数の .md 整理・各種 app/notebook 変更・Phase1 mask guard・私の動画修正）が含まれており、それらを一括退避してしまった。`git stash pop` は **EOL 正規化差分**（`git diff HEAD` はクリーンだが merge は生バイト差を拒否）で中断。

復旧: `git checkout 'stash@{0}' -- .` で stash の全ファイル内容を作業ツリーへ直接復元（マージ回避）。動画修正・Phase1 mask guard とも復元確認済み。**データ損失なし。stash@{0} はバックアップとして保持中**。

副作用: ユーザーの整理作業で削除されていた古い .md 11 個が HEAD 版として再出現している（破壊的操作のため自動削除せず、ユーザー判断に委ねる。内容は git HEAD に保全済みで再削除可能）。
