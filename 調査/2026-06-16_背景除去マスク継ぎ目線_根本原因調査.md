# 背景除去：マスク輪郭の「消える線」根本原因調査

- 調査日: 2026-06-16
- 対象: 動画背景除去で、シンバル・ドラム等の輪郭に沿って細い線（消える線）が入り合成が破綻する事象
- 区分: 調査のみ（実装変更なし）
- 関連: `調査/2026-06-15_ポイントプロンプト未反映_根本原因調査.md`（点プロンプト未反映の調査）

---

## 0. 結論（要約）

輪郭の細い線の正体は **transparent-background（背景透過AI）の内部処理ではなく、SAM2/SAMURAI が作る切り抜きマスク側** にある。具体的には次の二段の失敗が、最終 alpha に焼き付いている。

1. **複数マスクの統合（union）が二値ORで行われ、継ぎ目が揃わずすき間が残る。**
2. **その二値マスクの「ふくらませ（dilate）・ぼかし（feather）」を guard として tb 出力 alpha に乗算しており、継ぎ目処理が失敗した線がそのまま出力に残る。**

当初仮説（logit が 0 付近で振動してエイリアシング／重なり境界で AI の自信が落ちる）は、**コード上の直接的根拠が弱い推測**であり、主因ではないと判断する。

---

## 1. 調査対象コードと確定事実

### 1-1. transparent-background にマスクは渡していない

`pipelines/components/model_components.py`（TransparentBGExtractor.run, 約 641-644 行）:

```python
result = remover.process(Image.fromarray(crop_rgb), type="rgba", threshold=...)
```

- tb が受け取るのは **RGB のクロップ画像のみ**。マスクは入力されない。
- tb 自身は連続的でなめらかな alpha を返す。
- → **線は tb 内部処理では発生しない。**

### 1-2. マスクは「型抜き guard」として tb 出力に乗算される

`pipelines/components/model_components.py`（約 657-666 行）:

```python
if apply_mask_guard and mask.any():
    if mask_guard_feather > 0:
        guard = feather_binary_mask(mask, dilate_size=mask_guard_dilate, feather_radius=mask_guard_feather)
    else:
        guard = dilate_binary_mask(mask, kernel_size=mask_guard_dilate)
    full_alpha = full_alpha * guard
```

- guard は SAM2/SAMURAI の **二値 union マスク** を dilate / feather したもの。
- これを tb の連続 alpha に **乗算** する。
- → guard（マスク）に継ぎ目の線・すき間があれば、**そのまま最終 alpha に焼き付く。**

### 1-3. 統合は二値OR、継ぎ目処理は guard 段だけ

`pipelines/components/video_model_components.py`（SAM2VideoPropagator.run, 約 343-348 行）:

```python
obj_mask = (mask_logits.detach().cpu().numpy() > 0.0).squeeze()
union_mask = obj_mask if union_mask is None else (union_mask | obj_mask)
```

- 各対象（人・ドラム・シンバル）のマスクを **閾値 0.0 で二値化** し、**二値 OR** で統合。
- union の前段で SAM2 マスクに対するパディング/ぼかしは **一切行われない**。
- 縁をなめらかにする処理は 1-2 の guard 段（dilate/feather）が唯一。

---

## 2. 因果の流れ

1. 各対象のマスクを二値化（`> 0.0`）。
2. 二値 OR で 1 枚に統合 → 継ぎ目が揃わずすき間が残りうる。
3. 統合済み二値マスクを dilate/feather（guard 化）。継ぎ目のすき間を埋めきれないと細い線が残る。
4. その guard を tb の連続 alpha に乗算 → **継ぎ目の線が最終出力に焼き付く。**

tb 側（背景透過の下地）はきれいに抜けているのに、上から「継ぎ目入りの型紙」をかぶせて型抜きしている状態。

---

## 3. 当初仮説の棄却理由

| 当初仮説 | 判定 | 理由 |
| --- | --- | --- |
| logit が 0 付近で振動し単一エッジがエイリアシングして点線になる | 主因ではない | 二値化がロスなのは事実だが、本事象の主機構は「単一エッジの振動」より統合継ぎ目＋guard 処理失敗。コード上、線は統合・guard 段で生じる。 |
| 重なり境界で AI の自信が落ちて穴が空く | 根拠薄い | コードに「自信低下で境界に穴」を生む処理は存在しない。擬人的で曖昧。実体はマスク統合・整列の問題。 |

---

## 4. 残る別要因（独立）

- **奥行き順が無い**: 複数対象の前後（重ね順）を決めずに統合しているため、重なりの前後関係自体は別途未解決。これは継ぎ目線とは独立した課題（奥行き推定の検討事項）。

---

## 5. 改修方針（実装は別タスク）

優先順:

1. **（最優先）点を対象ごとに分けて追跡AIへ渡す**（点プロンプト未反映の主因。2026-06-15 調査参照）。
2. **複数マスクの統合と縁処理（dilate/feather）を作り直す**（継ぎ目の細い線を解消）。
3. **奥行き推定AIで前後の順番を決めて重ねる**（重なりの前後関係を正す。採用可否は事前計測）。

改修1・2が土台（点が出る＋継ぎ目がきれい）、改修3はその上の重ね順改善。改修3だけでは継ぎ目線は直らないため 1・2 を先行する。
