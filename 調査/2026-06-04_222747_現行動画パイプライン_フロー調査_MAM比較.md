# 現行 動画背景除去パイプライン フロー調査 ＋ Matting-Anything 比較報告

- **作成日時**: 2026-06-04 22:27:47
- **対象ファイル**: `gradio_app_sam2_transparent_BG_haystack_for_Movie.py`（UI/callback 層）/ `pipelines/sam2_tb_video_pipeline.py`（Pipeline 組み立て）/ `pipelines/components/video_model_components.py`（SAM2 伝搬・動画 I/O）/ `pipelines/components/model_components.py`（transparent-background 抽出）
- **比較対象**: `2026-06-04_222747_MattingAnything_BBOX-マスク-トラッキング-統合フロー調査.md`
- **目的**: 「DINO で BBOX は出る／SAM2 連携・マスク・トラッキング・マスク統合に不備があり、画面下のマスクが横一直線で切れる。パイプラインは壊れている」というユーザー観察の **コード上の根本原因**を特定する。
- **前提（ユーザー所見の確認）**: DINO の意味理解・BBOX 生成は機能している。問題は **BBOX → SAM2 → マスク → トラッキング → マスク統合**の配線にある。

---

## 0. 結論サマリ（先に要点）

1. **横一直線切れの真因は「SAM2 mask の外接矩形クロップ」**。`TransparentBGExtractor.run` は SAM2 mask の **形を使わず**、mask の外接矩形（≒ BBOX）で画像をクロップして transparent-background を適用し、その矩形範囲に貼り戻す。SAM2 mask が対象を囲みきれず外接矩形が途中で切れると、**矩形下端＝横一直線で alpha が切れる**。→ ユーザーの「BBOX そのものをマスク範囲に使っている疑い」は本質的に正しい。
2. **複合対象（ドラム＋人）が取れない配線**。`detect_text_boxes_for_video` は DINO 候補のうち **最上位 1 個（`boxes[0]`）だけ**を SAM2 prompt にコピーする。複数 box を選んで union する UI も callback もディスク上の現行ファイルに存在しない。
3. **Component 層は multi-box union / フレーム選択 / 双方向伝搬に対応済みだが、UI/callback 層が呼んでいない（配線欠落＝事実上の機能後退）**。`SAM2VideoPropagator.run` は `boxes` / `prompt_frame_idx` / `bidirectional` を受けて OR 統合できるのに、`run_video_background_removal` はこれらを渡さず単一 `box` のみ渡す。
4. **計画・記録（WHITEBOARD）とコードの乖離**。WHITEBOARD は複合対象 union・frame slider・tracker/background dropdown・tracking overlay を「完了」と記録するが、現行ファイルにこれらの UI/配線がなく、関連 **ユニットテスト 7 件が失敗**する。
5. オリジナル MAM とは別系統。現行動画版は **MAM の matting（連続 α）を使わず**、SAM2 二値 mask → transparent-background で抽出する。MAM 由来の不具合ではない。

## 1. 現行 動画パイプラインのフロー（コード実態）

```mermaid
flowchart TB
    subgraph UI[gradio movie UI / callback]
      V[input_video] --> EF[extract_first_frame\n第1フレーム→Prompt Canvas]
      EF --> PC[SAM2 Prompt Canvas\nクリックで box/point]
      TP[Text Prompt] --> DT[detect_text_boxes_for_video\nDINO で候補 bbox]
      DT -->|boxes[0] 最上位1個のみ| ST[prompt_state.box]
      PC --> ST
      ST --> RUN[run_video_background_removal]
    end
    RUN --> PIPE[build_sam2_tb_video_pipeline\n（propagator 注入なし）]
    PIPE --> VR[VideoReader\nframe サンプリング]
    VR --> SP[SAM2VideoPropagator\nbox 1個で伝搬]
    SP --> TB[TransparentBGVideoExtractor]
    SP --> OV[TrackingOverlayWriter]
    TB --> OUT[RGBA/alpha/preview 出力]
```

### 1.1 入口（callback）
- `run_video_background_removal(video_path, prompt_state, max_frames, frame_step, output_mode_label, rgba_codec, tb_mode, tb_jit, tb_threshold, tb_output_type, crop_padding, progress)`
  - **引数に `boxes` / `prompt_frame_idx` / `bidirectional` / `tracker_model` / `background_model` / `overlay_enabled` が無い**。
  - SAM2 へは `prompt_state` の **単一 `box`（または points）** のみを渡す。
- `get_video_pipeline()` は `build_sam2_tb_video_pipeline()` を **引数なし**で呼ぶ（propagator 注入経路は未使用）。

### 1.2 Text Prompt → BBOX（DINO）
- `detect_text_boxes_for_video(...)` は DINO で候補 bbox を出すが、**`boxes[0]`（最上位 1 個）を `state["box"]` にコピー**するだけ。
- 候補を複数選んで union する `CheckboxGroup` / `apply_selected_boxes` は **現行ファイルに無い**。
- UI の placeholder にも「Only first-frame prompt is supported in this version.」が残る。

### 1.3 SAM2 伝搬（Component 層は高機能）
- `SAM2VideoPropagator.run(frames, metadata, points, labels, box, boxes, object_id, prompt_frame_idx, bidirectional, progress_callback)`。
  - `boxes` 指定時: 各 box を obj_id 1..N で登録し、**frame ごとに全 obj の mask を OR 統合**。
  - union は **mask logits > 0**（`(mask_logits > 0.0)`）で生成。**BBOX を直接 mask にしてはいない**（このレイヤーは正しい）。
  - `bidirectional` で `prompt_frame_idx` 起点に forward/reverse 2 pass。
  - tracker 情報（`tracker_config`/`tracker_checkpoint`/`samurai_mode`）を masks metadata に記録。
- **しかし callback がこれらを渡さないため、実運用は常に単一 box・forward only・先頭フレーム起点**で動く。

### 1.4 透過抽出（横一直線切れの発生点）
- `TransparentBGVideoExtractor` → frame ごとに `TransparentBGExtractor.run(image, mask, ..., crop_padding)` を呼ぶ。
- `TransparentBGExtractor.run`（**核心**）:
  1. `bbox = mask_to_bbox(mask, padding=crop_padding, image_shape=...)` … SAM2 mask の **外接矩形**を取る。
  2. `crop_rgb = image_rgb[y_min:y_max, x_min:x_max]` … その矩形で **画像をクロップ**。
  3. `rgba_crop = remover.process(crop_rgb, type="rgba")` … **クロップ内に transparent-background を適用**（SAM2 mask の形は渡していない）。
  4. `full_alpha[y_min:y_max, x_min:x_max] = alpha_crop` … 矩形範囲だけに alpha を貼り戻す。矩形外は alpha=0。
- → **最終 alpha の範囲＝「SAM2 mask の外接矩形」**。SAM2 mask が対象を囲みきれず矩形下端が腰などで切れると、**その下端で alpha が横一直線に途切れる**。これがスクリーンショットの現象と一致。

### 1.5 トラッキング overlay
- `TrackingOverlayWriter` は追跡確認用 overlay を出せるが、callback が `overlay_enabled` を渡さず、UI に該当チェックも無い（未配線）。

## 2. 横一直線切れ：根本原因チェーン

```mermaid
flowchart LR
    A[DINO 複合対象\n人＋ドラム検出] --> B[detect_text_boxes_for_video\nboxes[0] 最上位1個のみ採用]
    B --> C[単一 box を SAM2 へ\nUI に複数box選択/union無し]
    C --> D[SAM2 mask が\n対象を囲みきれず下端切れ]
    D --> E[TransparentBGExtractor\nmask 外接矩形でクロップ]
    E --> F[矩形下端=横一直線で\nalpha が途切れる]
```

1. DINO は複合対象を検出できるが、callback が **最上位 1 box** しか使わない。
2. UI に **複数 box 選択・union 適用・フレーム選択** が無い（テストが要求する要素が欠落）。
3. SAM2 は単一 box 範囲でしかマスクを作れず、対象全体（全身＋ドラム）を囲めない。
4. `TransparentBGExtractor` が **SAM2 mask の外接矩形でクロップ**するため、mask が切れた範囲＝矩形でそのまま alpha が切れる。
5. 結果、**画面下が横一直線で切れる**。ユーザーの推測（「BBOX をマスク範囲に使っている」）はこの 4 段目で実際に起きている。

> 補足: `crop_padding` を上げても外接矩形が広がるだけで、**SAM2 mask 自体が対象を取りこぼしている根本**は解消しない。むしろ大きすぎると背景を巻き込む。

## 3. 計画・記録（WHITEBOARD）とコードの乖離

- WHITEBOARD は「複合対象 multi-box union（Phase A–E）」「frame slider」「双方向伝搬」「tracker/background dropdown」「tracking overlay UI」を **完了**と記録。
- しかし現行 `gradio_app_sam2_transparent_BG_haystack_for_Movie.py` には対応 UI/配線が無く、関連ユニットテストが **7 件失敗**（実測）:
  - `test_movie_app_exposes_text_prompt_to_box_flow`（`## 1. フレーム取得系` 等の見出し・`GroundingDINOMultiBoxDetector`・`STAGE_PROGRESS_RANGES` 不在）
  - `test_extract_first_frame_outputs_resets_prompt_slider`
  - `test_movie_app_exposes_frame_selection_and_union_controls`
  - `test_movie_app_exposes_tracking_overlay_ui`
  - `test_movie_app_exposes_tracker_dropdown`
  - `test_movie_app_tracker_dropdown_drives_pipeline_config`
  - `test_movie_app_exposes_background_model_dropdown`（`background_model` 不在）
- **解釈**: Component 層（`SAM2VideoPropagator` 等）は計画どおり実装済みだが、**UI/callback 層が旧版に巻き戻った（または未配線のまま）**。機能は「部品はあるが繋がっていない」状態。

> プロジェクト規約では「MD と出力が食い違う場合は MD を正」とするが、本件はユーザーが **現行ファイルの実態調査**を明示依頼しているため、ディスク上のコードを事実として報告する。乖離自体が最重要の発見である。

## 4. Matting-Anything（オリジナル）との比較

| 観点 | オリジナル MAM（静止画） | 現行 動画版（SAM2 + transparent-background） |
|---|---|---|
| プロンプト | text / point / box（単一） | text→box（最上位1個）/ Canvas の box・point |
| セグメンタ | SAM v1（image_encoder + mask_decoder） | SAM2 video predictor（フレーム伝搬） |
| マスクの使われ方 | SAM 二値 mask を **M2M のガイダンス**に使用 | SAM2 二値 mask の **外接矩形でクロップ**して transparent-background に使用 |
| 最終出力 | **M2M の連続 α マット**（髪・境界がソフト） | transparent-background の α（**矩形範囲に限定**） |
| 複合対象 union | **無し**（最高 confidence 1個） | Component は対応済みだが **UI 未配線で無効** |
| トラッキング | **無し**（静止画のみ） | SAM2 で対応（ただし frame 選択/双方向は **UI 未配線**） |
| 横切れリスク | M2M が α を画素単位で出すため低い | **外接矩形クロップで高い**（本件の主因） |

**要点**: 現行動画版は MAM の matting を継承しておらず、α の範囲が「SAM2 mask の外接矩形」に制約される設計。MAM の「連続 α を画素単位で出す」強みが失われている。

---

## 5. 推奨対応（優先度順、実装は別タスク）

1. **（最優先）外接矩形クロップへ SAM2 mask を適用**: `TransparentBGExtractor` で transparent-background の α に **SAM2 mask を AND**（または mask 内のみ残す）し、矩形外の直線切れを解消する。`SAM2GuardFilter`（mask 外 alpha 削り）は `model_components.py` に**実装済みだが現行 `sam2_tb_video_pipeline.py` に未接続**のため、これを透過抽出パイプラインに組み込む方向で検討。
2. **複合対象 union の UI 復旧**: DINO 候補を `CheckboxGroup` で複数選択 → `apply_selected_boxes` → `SAM2VideoPropagator(boxes=...)` の経路を callback に再配線（Component は対応済み）。
3. **frame 選択・双方向伝搬の UI 復旧**: `prompt_frame_idx` slider と `bidirectional` を callback 引数に追加し、`SAM2VideoPropagator` に渡す。
4. **tracker / background dropdown の復旧**: テストが要求する `tracker_model` / `background_model` を UI に復旧し pipeline config を駆動。
5. **回帰テストの GREEN 化**: `tests/unit/test_video_pipeline_wiring.py` の 7 失敗を配線復旧で解消し、計画と実装を一致させる。
6. **素材限界の周知**: 被写界深度ボケ・前景/背景同色は DINO/SAM2 が原理的に苦手。チームへ「素材側の制約」として共有（本件のパイプライン不具合とは別問題）。
