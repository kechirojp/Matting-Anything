## 背景

`Sam2_Transparent_Background_Haystack.py` / `.ipynb` と `gradio_app_sam2_transparent_BG_haystack.py` で、`Predict SAM2 Candidate Masks` と `Detect Text Boxes` の両方が初回だけでなく 2 回目も 5 分以上かかっている。Colab は GPU ランタイムの想定だが、Gradio プロセス内で CPU になっている可能性がある。動画マスク追跡ではこの遅さが致命的なため、静止画の SAM2 / GroundingDINO 両方を起点に、必要なら Movie 版にも横展開する。

2026-05-28 19:06 追記: 映像制作会社の実運用では CPU 実行は緊急回避専用。SAM2 / GroundingDINO / SAM2 video / legacy MAM Gradio の重い推論は既定で GPU 必須・CUDA 不可時 fail fast とし、意図的な CPU 回避運用だけ `MATTING_ANYTHING_ALLOW_CPU=1` で許可する。

## 調査方針

先に修正を決め打ちせず、**実行中 Gradio プロセス内の計測と device 証跡を SAM2 / GroundingDINO の両方で取ってから原因を分類する**。

## 原因別の対応方針

| 計測結果 | 原因候補 | 対応 |
|---|---|---|
| Gradio process で `cuda=False` | Colab runtime / torch wheel / subprocess environment | Notebook と Gradio process の device 診断を表示し、環境修正手順を提示 |
| SAM2 / GroundingDINO の `warm_up()` が毎回長い | predictor / detector / pipeline cache 不成立 | global cache / component lifecycle を修正 |
| checkpoint load が長い、Drive 上 | Google Drive I/O | Colab local disk へ checkpoint コピーする導線を検討 |
| `set_image()` が毎回長い | embedding 再計算 / 巨大画像 | same-image embedding cache または入力サイズ制御を検討 |
| CPU で bfloat16 autocast | CPU autocast 遅延 | CUDA の時だけ autocast、有効 device を明示 |
| GroundingDINO forward が長い | CPU fallback / CUDA custom ops 不在 / 巨大画像 | ERR004/ERR006/ERR010 系として修正 |
| SAM2 / GroundingDINO は速いが preview が長い | postprocess / overlay | preview 生成を軽量化または任意化 |
| Movie だけ長い | temp JPEG / video init / per-frame tb / writer | Movie pipeline の段階別最適化 |
