# DEVA方式 SAM2+BEN2/TB ハイブリッド動画アプリ: 使用モデル・チェックポイントサイズ・VRAM概算

- 調査日: 2026-07-01（配布対象アプリの変更に伴い同日更新）
- 対象: `gradio_app_sam2_ben2_tb_hybrid_deva_manual_for_Movie.py`（手動プロンプト版。
  使用モデル・チェックポイントは自動版 `gradio_app_sam2_ben2_tb_hybrid_deva_for_Movie.py` と同一）
- 注意: 開発用 `.venv` にはローカル GPU / torch 実行環境が無いため、VRAM 値は**実測ではなく
  各モデルの公称パラメータ数・重みサイズ・一般的な推論時オーバーヘッドからの概算**である。
  実機（想定: ローカル RTX 4090, `pyproject.toml` 記載）での実測が別途必要。

## 1. 使用モデル・チェックポイントサイズ一覧

| 役割 | モデル | 実装コンポーネント | チェックポイント | ファイルサイズ | 備考 |
|---|---|---|---|---|---|
| テキスト→bbox 検出 | GroundingDINO SwinT-OGC | `detection_island.py`（内部で GroundingDINODetector 相当） | `checkpoints/groundingdino_swint_ogc.pth` | **約 662 MB** | 実測（本リポジトリ配置ファイル） |
| 動画マスク伝搬 | SAM2.1 Hiera-Large | `SAM2VideoPropagator`（`video_model_components.py`） | `checkpoints/SAM2/sam2.1_hiera_large.pt` | **約 857 MB** | 実測。既定 config: `sam2.1_hiera_l.yaml` |
| 背景除去（人物領域） | transparent-background（InSPyReNet, base mode） | `TransparentBGExtractor`（`model_components.py`） | `checkpoints/transparent_BG/ckpt_base.pth`（未配置。初回実行時に自動DL） | **約 170〜180 MB**（base mode 概算。ローカル未DLのため実測不可） | `tb_mode="base"` が既定 |
| 背景除去（全画面） | BEN2 (Base) | `BEN2Extractor`（`ben2_components.py`） | `checkpoints/BEN2/BEN2_Base.pth`（PyTorch）/ `model.safetensors` | **約 1,082 MB**（.pth）/ **約 363 MB**（.safetensors） | 初回推論時に HuggingFace `PramaLLC/BEN2` から自動取得も可 |

**合計チェックポイントサイズ（実行に必要な分のみ、.pth系で計上）:**
662 MB (GroundingDINO) + 857 MB (SAM2.1-L) + 180 MB (TB base, 概算) + 1,082 MB (BEN2 .pth)
**≒ 約 2.78 GB**（BEN2 を safetensors 版で運用する場合は約 2.06 GB）

> 参考: リポジトリ内には GroundingDINO 重みの複製が6個（`.pth`, `.pth.1`〜`.pth.5`）存在するが、
> 実行で参照されるのは `checkpoints/groundingdino_swint_ogc.pth` の1個のみ。複製は配布物には不要。
> 同様に `mam_vitb/vith/vitl.pth`（各約31MB）は本アプリでは未使用（MAM本体専用）。

## 2. VRAM必要量の概算

### 2.1 モデル重み由来のVRAM（fp32 ロード時の目安）

| モデル | 重みサイズ | 推論時オーバーヘッド込み概算VRAM |
|---|---|---|
| GroundingDINO SwinT-OGC | 662 MB | 約 2.0〜3.0 GB（Swin-T backbone + テキストエンコーダ、1枚検出フレームのみ処理） |
| SAM2.1 Hiera-Large（動画伝搬） | 857 MB | 約 4.0〜7.0 GB（video state に全フレームの image embedding / memory bank を保持するため、動画長・解像度に応じて増加） |
| transparent-background base（InSPyReNet） | 約180 MB | 約 2.0〜3.0 GB |
| BEN2 Base | 1,082 MB（fp32相当） | 約 3.0〜4.5 GB |

### 2.2 アプリ全体としての概算

本アプリはパイプライン内で GroundingDINO → SAM2 → (BEN2 + transparent-background) の順に
呼び出すが、Gradio プロセスは一度ロードしたモデルをキャッシュして再利用するため、
**1回のアプリ起動〜1本の動画処理を通すと、実質的に4モデルすべてがGPU上に同時常駐しうる**。

- **保守的な概算合計: 約 12 GB 〜 18 GB VRAM**
  （動画の解像度・フレーム数・SAM2 の双方向伝搬有無で変動。4K・長尺・双方向伝搬ONだと
  SAM2 の memory bank 肥大化で上振れしやすい）
- 標準 SAM2 経路は本リポジトリでは **autocast 無効（fp32 維持、`autocast_dtype="none"`）**
  のため、fp16 運用時より VRAM 消費は大きめ（`config/inference_models.toml` 記載の設計判断）。
- SAMURAI トラッカーは本アプリでは未使用（`DevaSemiOnlineTracker` は標準 SAM2.1 経由の
  `SAM2VideoPropagator` を既定使用）。

### 2.3 推奨GPU目安

| GPU VRAM | 判定 |
|---|---|
| 24 GB（RTX 4090 等、`pyproject.toml` の想定環境） | 余裕あり。4K・長尺動画でも安定稼働見込み |
| 16 GB | 短〜中尺・FHD程度なら動作見込みだが、4K・長尺・双方向伝搬併用時は要注意 |
| 12 GB以下 | 非推奨。解像度ダウンサンプルや `per_object_logits_max_side` 等の縮小設定が必須級 |

### 2.4 注意事項・不確実性

- transparent-background の checkpoint は本リポジトリの `checkpoints/transparent_BG/` に
  未配置（空フォルダ）。実サイズは PyPI 配布パッケージの実測ではなく一般的な InSPyReNet base
  モデルサイズからの類推であり、要検証。
- 開発環境に GPU が無いため、本レポートのVRAM値はすべて**概算**。実機（RTX 4090等）での
  `nvidia-smi` 実測を別途行い、本レポートを実測値で更新することを推奨する。
- host-RAM 枯渇（VRAMではない）の既知不具合として `ERROR_LOG.md` ERR068（GroundingDINOが
  personを多数検出→per_object_logits全フレーム蓄積でRAM枯渇）があり、VRAMとは別にホスト
  メモリの余裕（16GB以上推奨）も必要。
